Project Description;

As the name of the project implies the purpose of this is to conduct a forensic investigation into 'Noise Storms' and in particular the LOVE Noise Storm that was first reported by GreyNoise in August 2024 as part of their StormWatch podcast https://www.greynoise.io/storm-watch-episodes/andrew-morris-on-noise-storms-uncovering-a-hidden-message-in-millions-of-icmp-packets.
A brief overview of what 'Noise Storms' are is that they "are large-scale spoofed packet events first observed in January 2020, coinciding with the U.S. military action against Iranian General Soleimani" In the case of this storm the packets used spoofed IP addreses originating from Brazil and target cloud or network service providers every few months by sending multiple ICMP ping requests at one device on their network. This storm gets its name because these packets contain the word "LOVE" in ASCII in the payload of the packet.

During the course of my forensic analysis of this phenomenon I have produced a detection script to be ran on packet captures to discover if there is any presense of these "LOVE" packets as well as a modified version of the program believed to be responsible for these packets and a parser for parsing packet captures and fixing the data capture to be readable by reversing the measures taken to hide the capture in the first place such as the modifying of the send timestamp on the program to hide it from dissectors when being viewed.
